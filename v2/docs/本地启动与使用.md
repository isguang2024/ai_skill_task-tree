# 本地启动与使用

本文档面向本地开发者和本地部署使用者。

## 1. 启动方式

### 方式 A：直接用批处理

- 后端：`C:\Users\Administrator\Desktop\Ai任务步骤记录\task-tree-v2\启动后端.bat`
- 前端开发：`C:\Users\Administrator\Desktop\Ai任务步骤记录\task-tree-v2\启动前端.bat`

### 方式 B：手动命令启动

后端：

```powershell
cd C:\Users\Administrator\Desktop\Ai任务步骤记录\task-tree-v2\backend
$env:TTS_ADDR="127.0.0.1:8880"
go run ./cmd/task-tree-service serve
```

前端开发：

```powershell
cd C:\Users\Administrator\Desktop\Ai任务步骤记录\task-tree-v2\frontend
npm run dev
```

## 2. 开发态与发布态的区别

### 开发态

- 前端地址：`http://127.0.0.1:5174/`
- 后端地址：`http://127.0.0.1:8880`
- 适合前端联调与热更新

### 发布态

先构建前端：

```powershell
cd C:\Users\Administrator\Desktop\Ai任务步骤记录\task-tree-v2\frontend
npm run build
```

再启动后端，然后浏览器访问：

- `http://127.0.0.1:8880`

发布态下，不需要单独启动前端 dev server。

## 3. 为什么不能直接打开 dist/index.html

`task-tree-v2` 不是纯静态离线页。页面会继续请求：

- `/v1/...`
- `/mcp`
- 其他同源接口

所以正确打开方式是“启动后端，再访问 `8880`”，不是双击 `index.html`。

## 4. DXT 的本地使用

V2 的 DXT 目录是：

- `C:\Users\Administrator\Desktop\Ai任务步骤记录\task-tree-v2\task-tree-dxt`

关键文件：

- `manifest.json`
- `proxy.mjs`
- `task-tree.dxt`

安装前请先确认本地服务已启动，并且 `proxy.mjs` 指向：

- `http://127.0.0.1:8880/mcp`

## 5. 常见入口

- 工作台：`http://127.0.0.1:8880`
- API：`http://127.0.0.1:8880/v1/...`
- MCP：`http://127.0.0.1:8880/mcp`
- 前端开发：`http://127.0.0.1:5174/`

## 6. 常见排查

### 页面打不开

先确认后端是否已启动，并检查：

```powershell
curl http://127.0.0.1:8880/healthz
```

### 前端开发页打不开

确认 `frontend` 下是否已执行：

```powershell
npm install
npm run dev
```

### Claude Desktop 扩展连不上

优先检查：

1. V2 服务是否已启动
2. `task-tree-v2/task-tree-dxt/proxy.mjs` 是否仍指向 `8880`
3. DXT 版本是否正确
